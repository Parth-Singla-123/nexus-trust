# 📊 Fraud Detection Platform

A production-grade, multi-layer fraud detection system combining **edge inference, real-time stream processing, and risk intelligence orchestration** to enable fast, adaptive, and scalable fraud prevention.

---

## 🚀 Overview

This platform is designed as a **multi-stage fraud decision system** that evaluates transactions across different time horizons:

- **Immediate (milliseconds)** → Edge inference in the browser  
- **Short-term (minutes to hours)** → Streaming behavioral analysis  
- **Long-term (days to weeks)** → Risk intelligence and user profiling  

Unlike traditional single-model systems, this architecture separates **prevention, monitoring, and enforcement**, enabling more accurate and timely fraud decisions.

---

## 🧠 Core Architecture

The system follows a **layered decision loop**, where each layer contributes unique intelligence:


Client (Next.js + TensorFlow.js)
↓
Edge Scoring (Real-time inference)
↓
API Layer (Validation + Persistence + Policy Enforcement)
↓
Stream Layer (PySpark - Behavioral Aggregation)
↓
Risk Intelligence (R - Scoring + Clustering)
↓
BI Layer (Power BI - Monitoring & Insights)


---

## ⚙️ How It Works

### 1. Edge Decisioning (Client-Side)

- Captures transaction intent and contextual signals
- Performs **real-time fraud scoring using TensorFlow.js**
- Blocks high-risk transactions instantly (sub-second latency)
- Reduces backend load by filtering obvious fraud early

---

### 2. API Layer (Business Logic & Enforcement)

- Validates and persists transactions
- Maintains account state
- Streams events for downstream processing
- Enforces blacklist decisions generated by risk intelligence

---

### 3. Stream Processing (PySpark)

Processes live transaction data to generate behavioral signals:

- Rolling transaction windows (10 minutes, 24 hours)
- Spending patterns and frequency anomalies
- Rule-based risk escalation

---

### 4. Risk Intelligence (R)

Applies advanced analytics on enriched data:

- Fraud class rebalancing for rare event detection
- K-means clustering for hidden fraud cohorts
- Weighted user risk scoring
- Blacklist generation for enforcement

---

### 5. BI Layer (Power BI)

- Provides a **fraud operations dashboard**
- Tracks fraud trends, hotspots, and user risk levels
- Enables data-driven decision-making

## 🧮 Detection Strategy

### Edge Model

**Inputs:**
- Transaction amount  
- Hour of transaction  
- Device risk score  
- Location risk score  
- Transaction velocity  

**Scoring Function:**


z = 0.00002·amount + 0.8·I(hour ≤ 4) + 1.4·device_risk + 1.2·location_risk + 0.15·txn_velocity

p = σ(z - 2.2)


**Policy:**
- Block if `fraud_probability ≥ 0.82`

---

### Stream-Level Rules (PySpark)

Triggers risk escalation when:

- High edge fraud probability (`≥ 0.8`)
- Rapid transaction bursts (`≥ 4 transactions / 10 min`)
- High-value suspicious transactions (`> 250,000 at night`)
